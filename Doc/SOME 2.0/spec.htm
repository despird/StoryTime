2.1 Common Rules

l          In Sequence definition, every statement will be ended with “;”,if you place a pair of curly brackets, that only means you want to make a definition of a method. In this situation, you can place “{“ or “}” anywhere, but a “;” must not appear before “{“ ,but must appear after “}”.

l          You may start with any method of any class that you’ve defined in some other place, each definition seems just like a sequence diagram in UML

l          If you start a new definition, both the type and the calling method must have been declared in Type declaration area

3. Common Rules for all

l          This language contains no keywords

l          Only const string and digit can be resolved as constant variables

l          Unknown type is accepted and will be reserved in the target code.
